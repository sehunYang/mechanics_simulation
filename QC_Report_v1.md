## 4. 정적 코드 리뷰 (물리 로직)

### 🔴 치명 (Critical)

**C1. 질량 0 입력 → NaN 전파, 복구 불가** `[재현됨 → 수정 완료]` — `_numInput`(4315행)이 `min` 속성만 설정하고 입력값을 클램프하지 않아, 질량 칸에 "0"을 직접 타이핑하면 `mass=0`이 그대로 저장됨. 시뮬 전체에 NaN 가드가 없어 질량 0인 물체가 충돌하는 순간 충돌 해소식(3921~3984행)의 `Infinity/Infinity`가 NaN을 만들고, 실·도르래로 연결된 모든 물체로 전파됨. **재현 결과**: 질량 0 원 2개 충돌 → 4초 만에 두 물체 모두 physX/vx = NaN. 초기화 버튼은 질량 0이 든 스냅샷을 복원하므로 값을 고치기 전까지 반복 발생.

### 🟠 주요 (Major)

**M1. 사각형이 곡면 바닥(ARC)을 관통** `[재현됨 → 수정 완료]` — `_resolveRectFloor`(3383행)에 원형(`_resolveCircleFloor` 3312행에는 있음)과 달리 정점(joint) 처리가 없음. 사각형 모서리-선분 검사만으로는, 호의 꼭짓점이 사각형 **밑변 안쪽**을 뚫고 들어오는 경우(모서리들은 양옆 경사면 평면보다 위에 있어 미검출)를 놓침. **정확한 재현 조건**: 꼭짓점~모서리 접촉면 간극이 사각형 높이보다 클 때 — 예: 폭 20 사각형을 ARC_UP 위에 낙하 → 바닥을 그대로 통과해 자유낙하(-431m). 폭 1~3에서는 모서리 검사가 잡아내므로 관통 대신 약간의 침하(0.06m)·잔진동만 발생.
  - *참고*: 최초 재현 실험(폭 1 사각형, y=-18.6 낙하)은 궤적 재분석 결과 관통이 아니라 **마찰 없는 볼록면에서 미끄러져 바닥 끝으로 떨어진 정상 물리**였음. 실제 관통은 위 폭 20 조건에서 A/B 테스트로 확증.

**M2. 유효성 검사 범위 협소** — `validateAll`(4084행)은 "도르래에 실 1개만 연결"만 검사. 삭제된 요소에 걸린 실(제약이 조용히 무시됨), 초기 겹침 배치, 이웃 없는 용수철 등은 경고 없이 실행됨.

## 5. 수정 내역 (2026-07-05)

### C1 수정 — 질량 0/음수/NaN 차단 (2중 방어)

1. **`_numInput` 입력 클램프**: 타이핑 입력값도 min/max 범위로 클램프해 `onChange`에 전달. 포커스 아웃(change) 시 필드 표시값도 정리. 스피너만 막던 `min` 속성의 허점 제거.
2. **`initPhysics` 질량 가드**: 시뮬 시작 시 rect/circle의 질량이 `> 0`이 아니면(0·음수·NaN 포함) 0.1로 보정. 과거에 저장된 오염 스냅샷이 복원되어도 NaN이 발생하지 않음.

**검증**: 질량 칸에 "0"·"−5" 타이핑 → 0.1로 클램프 확인. 질량 0을 코드로 직접 주입 후 충돌 시뮬 4초 → 전체 유한값 유지, 충돌 정상 해소.

### M1 수정 — `_resolveRectFloor`에 정점(joint) 보정 단계 추가

원형 충돌(`_resolveCircleFloor` 1-2단계)과 동일한 취지로, 바닥 미세 선분의 연결점이 사각형 내부에 들어오면 최소 이동 축(MTV)으로 밀어내는 단계를 추가. 배출 방향은 해당 세그먼트 법선과 같은 쪽만 허용해 표면 반대편으로 튕기는 것을 방지.

**검증 (A/B 테스트, 동일 장면·구버전 함수 주입 비교)**:

| 장면: 폭 20 사각형을 ARC_UP 위 낙하 (10초) | 수정 전 | 수정 후 |
|---|---|---|
| 최저 y | **−431.2 (완전 관통, 자유낙하)** | 47.401 (꼭짓점 위 안착) |
| 최종 속도 | −98 m/s | 0 m/s |

추가 확인: 폭 3 사각형 꼭짓점 안착 시 침하 0.06m·잔진동(구버전) → 정확히 47.401 안착·vy=0(신버전)으로 접촉 품질도 개선. ELBOW 바닥 낙하 관통 없음.

## 6. 관찰 사항 (버그 아님, 참고용)

1. **월드 경계 없음**: 유한한 바닥을 벗어난 물체는 영원히 낙하하며 시뮬레이션에 계속 남음 (T13). 장시간 실행 시 좌표값만 커질 뿐 NaN은 발생하지 않으나, "화면 밖 이탈 시 제거/경고" 처리를 고려할 수 있음.
2. **기본 반발계수 e=1.0**: 기본값이 완전 탄성이라 물체가 영원히 튕김. 교육 목적이라면 의도된 설계일 수 있음.
